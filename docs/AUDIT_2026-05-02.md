## 2. New findings

### Performance

#### P2
- **`/api/recent-orders` unbounded list + N+1 fetch** — `src/app/api/recent-orders/route.ts:18-49`. `kv.list({ prefix: 'order:recent:' })` has no `limit` and individual `kv.get()` calls run sequentially in the loop. With 24h TTL and current order volume this is fine, but as orders scale (>200/day) this becomes a measurable latency drag on every homepage social-proof toast. Fix: pass `limit: 100` to `kv.list`, replace the for-loop with `Promise.all(list.keys.map(...))`. Original audit P2 — still applies.
- **`/api/search` calls `getProducts()` on every query** — `src/app/api/search/route.ts:134`. `getProducts()` fetches all 250 products with full descriptions, prices, and images on every search. Search has 20-req-per-IP-per-min rate limit so worst case is bounded, but for a typical search volume the route does ~30× the GraphQL work it needs. Fix: cache the product list in-memory across worker isolates (Workers Cache API or KV with 5-min TTL), or write a search-specific lightweight GraphQL query that only fetches `handle`, `title`, `description`, `image[0]`.
- **`/api/search` has no `Cache-Control` on responses** — same file, line 231. Repeated queries from the same browser get full Sanity + Shopify round-trips. Add `Cache-Control: private, max-age=30` on successful responses.

### Security

#### P1
- **`/api/search` query length not capped** — `src/app/api/search/route.ts:125`. The first audit flagged this; not yet addressed. Current code wraps single-word queries with wildcards (`*${query.trim()}*`) before passing to GROQ as a parameter. GROQ `match` on a parameterised value is safe from operator-injection, but a 10MB POST body or a query like `*****...****` can still exhaust Sanity's parser. Fix: cap query at 80 chars and strip non-alphanumeric/space characters before constructing the wildcard pattern.

#### P2
- **Klaviyo error string still leaks "Klaviyo API key not configured"** — `src/app/api/contact/route.ts:118` and `src/app/api/klaviyo-signup/route.ts:80`. Returns the raw configuration error string in a 500 response, fingerprinting the stack to anyone hitting the endpoint without env vars set. Low real-world risk in production (env is always set), but an attacker probing for misconfiguration gets a free signal. Fix: return generic `'Server configuration error.'` like the trade routes already do.
- **Webhook audit log entry doesn't include payload hash** — `src/app/api/webhooks/shopify/route.ts:314`. The `webhook:log:{topic}:{ts}` KV entry only logs `{ topic, received_at }`, so the audit trail can't help debug "did we actually receive this specific order?" without re-fetching from Shopify. Fix: include the order ID (`p.id`) or a SHA-256 hash of the payload. Not security-critical; observability improvement.

### SEO

#### P2
- **Search-route slug links missing trailing slashes** — `src/app/api/search/route.ts:183, 202, 221` build URLs like `/field-manual/equipment/${slug}` without trailing `/`. With `trailingSlash: true` in `next.config.ts`, every search-result click gets a 308 redirect. Same issue Batch 19 fixed for the Header dropdown; missed here. Fix: append `/` to all three.

## 3. What's improved since first audit

The codebase is in **substantially better shape** than the snapshot at `feat/p2-batch-16`:

- **Security posture:** zero JSON-LD injection points, defence-in-depth on PortableText `javascript:` URIs, strict character allowlist on Klaviyo filter substitutions, trade PIN regex tightened, referral codes capped, email domain DNS validation now blocks fake-domain submissions. The remaining P1 (search query cap) is a known item from the first audit.
- **Privacy posture:** every analytics call site is now consent-gated. Sentry scrubs PII on both server and client. Newsletter signup verifies consent server-side. Contact-form profiles are demonstrably transactional (Article 6/7 defensible). Age-gate query-string bypass removed.
- **Performance posture:** universal CSS transition gone, hero ships one image instead of three, OG images use Cloudflare flexible variants, service worker no longer caches stale chunk-referencing HTML, KlaviyoScript is a static stub.
- **Resilience posture:** branded error boundary, Sentry now actually fires from edge-runtime API routes, webhook handler returns 500 so Shopify retries, cart race condition fixed via `useRef`, transient Shopify outages no longer empty real users' carts.
- **Code hygiene:** zero `: any` types, zero TODO markers, zero remaining old-Sanity-client imports, dead `CountdownTimer.tsx` gone.

The remaining findings are mostly P2 polish — the audit list as it stood is essentially closed.
